# Предложение по улучшению архитектуры проекта

Цели:
- Ввести ясное разграничение ответственности между слоями приложения.
- Упростить масштабирование и тестирование компонентов.
- Обеспечить удобную структуру для дальнейшего рефакторинга.

Предлагаемая начальная структура:
- src/
  - core/        # Ядро приложения: доменные модели, контракты, общие интерфейсы
  - services/    # Службы: реализация бизнес-логики, взаимодействие между компонентами
  - handlers/    # Обработчики запросов / адаптеры (API, CLI, workers)
  - utils/       # Утилиты и вспомогательные функции

Дальнейшие шаги:
1. Перенести доменные модели и интерфейсы в src/core.
2. Выделить бизнес-логику в src/services и покрыть юнит-тестами.
3. Организовать адаптеры ввода/вывода в src/handlers.
4. Обновить CI/CD для новой структуры (пути сборки и тестов).
5. Постепенно рефакторить существующий код, сохраняя обратную совместимость через адаптеры.

Этот MR содержит только документацию и каркас директорий, чтобы начать обсуждение и планирование следующего этапа работ.
